def create_app():
    app = Flask(__name__)
    app.config.from_object('app.secure')
    app.config.from_object('app.setting')
    # 调用蓝图
    register_blueprint(app)

    # 直接调用核心对象app会报错
    db.init_app(app)
    db.create_all()
    return app

错误信息：
    'No application found. Either work inside a view function or push'
RuntimeError: No application found. Either work inside a view function or push an application context. See http://flask-sqlalchemy.pocoo.org/contexts/.
解决办法：
    db.create_all(app=app)


二、该Base为基类模型
class Base(db.Model):
    status = Column(SmallInteger, default=1)

错误信息：
    sqlalchemy.exc.ArgumentError: Mapper Mapper|Base|base could not assemble any primary key columns for mapped table 'base'
原因：
    Base直接继承的db.Model，所以sqlalchemy会以为Base类要创建表，但是由于Base只是一个基类，我们不希望它创建一个表，只需要继承它的子类创建表就可以了。
解决办法：
    class Base(db.Model):
        __abstract__ = True
        status = Column(SmallInteger, default=1)
    此时sqlalchemy就不会以为Base类要创建表了。
